# Changelog (작업 기록)

브랜치·PR 단위로 "이번에 한 일"을 남겨, 다음 세션·다른 역할이 **이전에 뭘 했는지** 복원할 수 있게 한다.

---

## 규칙

- **작업 요약**은 이 브랜치/PR에서 변경한 파일·결정 사항·이유를 담는다.
- **파일명**: `{브랜치명 또는 일감식별자}_{날짜 YYYYMMDD}.md` (예: `feature_programmer_item_20250301.md`). 브랜치명에 슬래시가 있으면 언더스코어로 치환.
- **위치**: 이 디렉터리(`docs/changelog/`)에 md 파일로 저장. PR description에 같은 내용을 적어도 되고, 머지 후 이 폴더에 복사해 두면 이후 검색·복원이 쉬움.
- **작업 시작 전**: 같은 브랜치의 이전 changelog·[CurrentStructure.md](../plan/CurrentStructure.md)를 읽고, 기존 코드·구조와 맞는 방향으로 진행한다.

---

```markdown
## 요약
(한두 문장)

## 변경 파일
- (경로·파일 목록)

## 결정 사항
- (무엇을 어떻게 하기로 했는지)

## 이유·비고
- (선택)
```
