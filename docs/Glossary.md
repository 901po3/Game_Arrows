# Glossary

프로젝트에서 공통으로 쓰는 용어 정의. 기획·개발·QA가 같은 의미로 사용할 것.

| 용어 | 정의 |
|------|------|
| **스테이지** | 게임 내 한 번의 플레이 단위(레벨/맵). 클리어 조건·보상이 정의됨. |
| **아이템** | 플레이어가 획득·사용하는 게임 내 오브젝트. 인벤토리·효과·수량 등이 있음. |
| **상품** | 스토어 등에서 구매 가능한 것. 유료/무료, 인앱 구매·광고 연동 등 포함. |
| **밸런스** | 난이도·보상·경제·성장 곡선 등 게임 플레이가 공정하고 재미있게 유지되는 상태. |
| **게임 정체성** | Arrow Puzzle의 핵심 루프·느낌·타겟 경험. 기획 변경 시 유지해야 할 기준. |
| **Plan** | 작업 전 작성하는 설계/기획 문서. 범위·방법·리스크를 담고 Plan 리뷰어 검토 대상. |
| **TODO** | docs/TODO.md에 기록하는 일감·개선·이슈. PM이 묶고 기획/개발에 배정. |
| **리소스** | 게임에 사용하는 미디어·에셋(이미지, 사운드, 애니메이션, 폰트 등). 구하는 방법·배치는 [resources/Resources.md](resources/Resources.md) 참고. |
| **에셋** | Unity 프로젝트 내 에셋(스크립트·이미지·사운드·프리팹 등). 리소스는 그중 게임에 쓰는 미디어를 넓게 이르는 말. |

(프로젝트가 진행되면 용어를 추가·수정한다.)
