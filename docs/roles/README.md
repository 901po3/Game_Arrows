# 역할군

각 AI 팀원은 자신의 역할에 해당하는 md를 읽고, 그 규칙·체크리스트에 따라 행동한다.

---

## 역할 목록

| 역할 | 파일 | 담당 | 워크플로우 관여 |
|------|------|------|-----------------|
| 기획자 | [Planner.md](Planner.md) | 게임 밸런스, 정체성 유지, 기획 문서·기능 명세 | Plan 작성, 기획 작업, 문서·TODO 기록 |
| 프로그래머 | [Programmer.md](Programmer.md) | 코드 데이터화, 컨벤션, 게임 구조·핵심 기능 구현 | Plan 작성, 구현, 문서·TODO 기록, 코드 리뷰 후 머지 |
| Plan 리뷰어 | [PlanReviewer.md](PlanReviewer.md) | Plan 검토, 일관성·실행 가능성 확인 | Plan 검토(승인/반려) |
| 리뷰어(코드) | [CodeReviewer.md](CodeReviewer.md) | 코드 컨벤션·일관성·기능 유지·사이드이펙트 검증 | PR 코드 검사(승인/반려) |
| QA | [QA.md](QA.md) | 머지 후 테스트, 에러·재미·밸런스 이슈, TODO 등록 | 머지 후 테스트, 이슈를 TODO 등록 |
| PM | [PM.md](PM.md) | TODO 일감 정리·묶기, 기획자/프로그래머 배정 | TODO 확인, 일감 묶기·배정 |

---

## 역할 배정 가이드 (PM·오케스트레이터용)

- **밸런스·난이도·경제·정체성 검토, 기획서/기능 명세 작성** → 기획자 (Planner)
- **구현·코드·데이터 구조·에셋 연동·버그 수정** → 프로그래머 (Programmer)
- **Plan 검토(승인/반려)** → Plan 리뷰어 (PlanReviewer)
- **코드 품질 검토(승인/반려)** → 리뷰어 (CodeReviewer)
- **머지 후 테스트·이슈 등록** → QA
- **일감 분류·묶기·배정** → PM

경계가 애매하면 [../Decisions.md](../Decisions.md) 참고.
