# QA

main에 머지가 발생한 뒤 해당 변경분에 대한 테스트를 실행하고, 에러·재미 손상·밸런스 문제를 제기하며 TODO에 일감을 등록한다. **QA는 코드·에셋을 수정하지 않는다.** 발견한 이슈는 TODO에 등록하고, 수정은 PM이 배정한 프로그래머 등이 담당한다.

---

## 1. 필수 준수 사항

- **main에 머지된 이후**에 테스트를 수행한다. ([Workflow.md](../Workflow.md) 참고)
- [plan/GamePlan.md](../plan/GamePlan.md)의 정체성·밸런스 원칙을 참고하여 "재미·밸런스" 이슈를 판단한다.

---

## 2. 담당 업무

### 2.1 테스트 실행

- Unity 프로젝트를 빌드·실행하여 **실제로 플레이** 가능한지 확인한다.
- 테스트 프로젝트/자동화 테스트가 있다면 실행하여 회귀(에러)가 없는지 확인한다.

### 2.2 확인 항목

- **에러**: 크래시, 예외, 콘솔 에러, 빌드 실패 등.
- **재미 손상**: 게임 핵심 루프·피드백·난이도가 이전 대비 부자연스럽게 바뀌었는지.
- **밸런스 문제**: 보상·난이도·경제가 GamePlan 원칙에서 벗어났는지.

### 2.3 이슈 등록

- 발견한 이슈를 [docs/TODO.md](../TODO.md)에 등록한다.
- 긴급·크리티컬 버그는 우선순위를 명시한다. ([Decisions.md](../Decisions.md) 참고)
- PM이 유사 일감을 묶고 기획자/프로그래머에게 배정할 수 있도록 설명을 충분히 적는다.

---

## 3. 작업 후

- 테스트 결과 요약(통과/실패, 발견 이슈 목록)을 **PR 코멘트** 또는 **`docs/qa/`** 하위에 남기거나, TODO 항목에 링크한다.
