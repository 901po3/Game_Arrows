# Plan 리뷰어 (Plan Reviewer)

Plan의 명확성·일관성·실행 가능성을 검토하고 승인 또는 반려한다.

---

## 1. 필수 준수 사항

- [Workflow.md](../Workflow.md)에 따라, **Plan이 제출된 후** 검토를 수행한다.
- **Plan 제출 위치·형식**: Plan은 PR description 또는 `docs/plan/` 내 문서(링크)로 제출한다. 팀에서 정한 방식으로 통일한다.
- **검토 결과**: 승인/반려와 사유는 **PR 코멘트**에 남긴다. 작성자는 그에 따라 작업 진행 또는 Plan 수정 후 재제출한다.
- 반려 시 사유를 명확히 적어 작성자가 수정할 수 있게 한다. ([Decisions.md](../Decisions.md) 참고)

---

## 2. 검토 기준

### 2.1 명확성

- 목표·범위·산출물이 구체적으로 적혀 있는가?
- 기획자·프로그래머가 동일한 의미로 이해할 수 있는가?

### 2.2 일관성

- [plan/GamePlan.md](../plan/GamePlan.md), [Glossary.md](../Glossary.md)와 용어·정의가 맞는가?
- 기존 Plan·문서와 충돌하는 내용은 없는가?

### 2.3 실행 가능성

- 제안된 방법이 현재 프로젝트 구조·기술 스택으로 구현 가능한가?
- 리스크·예외 케이스가 적어도 간단히 언급되어 있는가?

### 2.4 적정 범위

- 한 번에 다루기 적당한 크기인가? 너무 크면 나누도록 권장한다.

---

## 3. 검토 후 동작

- **승인**: 코멘트로 "Plan 승인"을 남기고, 작성자는 작업 진행 가능.
- **반려**: 반려 사유와 수정 요청 사항을 적고, 작성자는 Plan 수정 후 재제출.
