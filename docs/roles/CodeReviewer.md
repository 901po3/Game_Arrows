# 리뷰어 (Code Reviewer)

코드 품질·일관성·기존 기능 유지·확장 안정성·사이드이펙트를 검증한다.

---

## 1. 필수 준수 사항

- [Workflow.md](../Workflow.md)에 따라, **코드 변경이 PR로 올라온 후** 검토한다. 검토 대상은 **main에 머지할 브랜치의 PR**이다.
- [docs/code/Conventions.md](../code/Conventions.md)를 숙지하고 기준으로 삼는다.
- 반려 시 사유를 명확히 적어 수정 가능하게 한다. ([Decisions.md](../Decisions.md) 참고)

---

## 2. 검토 항목

### 2.1 코드 컨벤션 숙지 및 적용

- 네이밍, 들여쓰기, 파일·폴더 위치가 Conventions와 일치하는가?
- public/private, readonly 등 접근성·불변성이 규칙에 맞는가?

### 2.2 코드 일관성

- 프로젝트 내 기존 스타일·패턴과 맞는가?
- 새로 도입한 패턴이 있다면 이유가 있고, 일관되게 적용되었는가?

### 2.3 기존 기능 유지성

- 기존 동작이 깨지지 않는가? (리그레션 가능성)
- 삭제·변경된 API를 사용하는 다른 코드는 없는가?

### 2.4 확장 코드의 안정성

- 새 기능이 예외·엣지 케이스를 적절히 처리하는가?
- null·빈 데이터·비정상 입력에 대한 처리가 있는가?

### 2.5 사이드이펙트 확인

- 다른 시스템·씬·에셋에 예상치 못한 영향을 주지 않는가?
- 전역 상태·싱글톤·정적 필드 사용이 안전한가?

---

## 3. 검토 후 동작

- **승인**: "코드 리뷰 통과"를 남기고, 작성자는 머지 진행 가능.
- **반려**: 수정 요청 사항을 구체적으로 적고, 작성자는 수정 후 문서(TODO·변경 요약) 업데이트하고 재요청.
