# Arrow Puzzle — 기술 및 규칙 명세서

상위 기획: [GamePlan.md](GamePlan.md)

이 문서는 Arrow Puzzle의 핵심 메커니즘, 데이터 구조, 충돌 판정, 그리고 절차적 맵 생성(Procedural Generation) 알고리즘을 타 언어/엔진에서 동일하게 구현할 수 있도록 설명합니다.

---

## 1. 보드 및 그리드 시스템 (Grid & Board)

- **좌표계**: 2D 그리드 시스템을 사용합니다. 좌측 상단이 (0, 0)이며 우측 하단이 (size-1, size-1)입니다.
- **마스크 (Masking)**: 모든 맵이 정사각형은 아닙니다. `boolean[][] mask` 배열을 통해 특정 타일을 활성화/비활성화합니다.
  - 비활성화된 타일(빈 공간)에는 블록이 생성되지 않지만, 블록이 빠져나갈 때 통과할 수는 있습니다.
- **난이도별 보드 크기**:
  - **일반 (Lv 1~9)**: 4×4 → 5×5 → 6×6으로 점진적 증가. 정사각형 마스크.
  - **어려움 (10단위 레벨, 20 제외)**: 7×7 크기. 네 모서리가 잘린 십자가(Plus) 형태의 마스크.
  - **매우 어려움 (20단위 레벨)**: 9×9 크기. 네 모서리가 2×2 크기로 잘린 둥근 원형 형태의 마스크.

---

## 2. 블록(선) 데이터 구조 (Block Structure)

각 블록은 체스말처럼 한 칸을 차지하는 것이 아니라, 여러 칸을 차지하는 **뱀(Snake)**과 같은 연결된 리스트 구조입니다.

- **cells**: 블록이 차지하는 그리드 좌표들의 배열 `[{x, y}, {x, y}, ...]`
  - 반드시 상하좌우로 인접한 칸들로만 연결되어야 합니다.
  - 최소 길이는 2칸 이상입니다. (1칸짜리 점 형태의 블록은 존재하지 않음)
- **head (머리)**: `cells` 배열의 마지막 원소(`cells[cells.length - 1]`)가 항상 화살표가 달린 머리 부분입니다.
- **headDir (방향)**: 머리가 가리키는 방향입니다. (up, down, left, right)
- **state**: 블록의 현재 상태 (idle: 대기, flying: 빠져나가는 중, removed: 맵에서 제거됨)

---

## 3. 핵심 이동 규칙 (Slithering Movement)

이 게임의 가장 중요한 특징은 블록이 통째로 평행 이동하는 것이 아니라, 자신의 몸통을 따라 스르륵 빠져나간다는 것입니다. (뱀의 이동 방식과 동일)

- 화살표(머리)는 `headDir` 방향으로 직진합니다.
- 머리 뒤에 달린 나머지 몸통(`cells`의 앞부분)은 머리가 지나간 경로를 정확히 따라갑니다.
- 따라서 블록이 'ㄷ', 'ㄹ' 자로 꺾여 있더라도 주변의 다른 블록을 건드리지 않고 자신의 선을 따라 맵 밖으로 빠져나갑니다.

---

## 4. 충돌 및 이동 가능 판정 (Collision Detection)

블록이 자신의 선을 따라 빠져나가기 때문에, 충돌 판정은 블록 전체를 밀어보는 것이 아니라 **머리(Head)에서 뻗어나가는 직진 레이캐스트(Raycast)**로만 검사합니다.

### 탈출 가능 조건 (Is Block Free?)

- **시작점**: 블록의 머리 좌표 (head.x, head.y)
- **방향**: 화살표의 방향 (dx, dy)
- **검사 로직**: 머리에서 출발하여 맵 바깥으로 완전히 나갈 때까지 해당 직진 경로 상의 칸들을 1칸씩 검사합니다.
- **차단 조건**: 검사하는 칸에 아직 빠져나가지 않은(상태가 idle인) "다른" 블록의 몸통이 겹쳐 있다면 탈출 불가(막힘) 상태입니다.
- 빈 공간, 비활성화된 마스크 공간, 이미 빠져나간 블록의 공간은 자유롭게 통과할 수 있습니다.

### 의사 코드 (Pseudo-code)

```
function isBlockFree(block, allBlocks) {
    let dx = block.dir.x; // 1, -1, 0
    let dy = block.dir.y; // 1, -1, 0
    let head = block.cells[block.cells.length - 1];

    // 현재 맵에 남아있는 다른 블록들만 필터링
    let otherBlocks = allBlocks.filter(b => b.id !== block.id && b.state === 'idle');

    // 머리 앞칸부터 맵 끝을 넘어갈 때까지 직진 검사
    for (let step = 1; step <= MAX_BOARD_SIZE; step++) {
        let checkX = head.x + (dx * step);
        let checkY = head.y + (dy * step);

        for (let other of otherBlocks) {
            for (let cell of other.cells) {
                if (cell.x === checkX && cell.y === checkY) {
                    return false; // 다른 블록에 가로막힘!
                }
            }
        }
    }
    return true; // 무사히 맵 밖으로 빠져나갈 수 있음!
}
```

## 5. 자동 생성 알고리즘 규칙 (Procedural Generation)

퍼즐이 항상 100% 클리어 가능하고 빈틈없이 꽉 차도록 만드는 생성 알고리즘의 순서입니다. 엔진 포팅 시 이 로직을 구현해야 무한 모드가 가능합니다.

1. **공간 분할 (Partitioning)**
   - 마스크가 적용된 보드의 모든 칸을 추적합니다.
   - 무작위 칸에서 시작하여 빈칸을 무작위 보행(Random Walk)으로 연결해 길이 2~5칸짜리 선을 만듭니다.
   - 보드의 모든 칸이 꽉 찰 때까지 반복합니다.

2. **1칸짜리 블록 병합 (Merge Single Cells)**
   - 분할 결과 1칸짜리 고립된 블록이 생겼다면, 상하좌우에 있는 인접한 다른 블록의 머리나 꼬리에 강제로 편입시켜 선을 연장합니다. (모든 블록이 최소 2칸 이상이 되도록 보장)

3. **방향 할당 및 검증 (Assign Directions & Solver)**
   - 각 블록의 양 끝점 중 하나를 머리(head)로 설정하고 방향을 지정합니다.
   - **자가 겹침 방지**: 화살표가 자기 자신의 몸통을 향해 가리키면 안 됩니다.
   - **역추적 시뮬레이터**: 무작위로 블록들을 하나씩 맵 밖으로 빼보는 시뮬레이션을 돌립니다.
   - 모든 블록이 하나씩 차례대로 빠져나가 보드를 텅 비울 수 있는 **논리적으로 완벽한 해답(Solution)**이 존재하는 방향 조합을 찾을 때까지 조합을 찾습니다.
   - 해답을 찾으면 그 방향 설정을 최종 맵 데이터로 반환합니다.

---

## 6. 게임 루프 및 상호작용 (Game Loop)

- **터치/클릭**
  - `isBlockFree`가 true일 때: 블록 상태를 `flying`으로 변경하여 애니메이션을 재생하고, 지정된 시간 후 보드에서 완전히 제거(`removed`)합니다.
  - `isBlockFree`가 false일 때: 블록을 좌우로 흔드는 오류 애니메이션을 재생하고 생명을 1 감소시킵니다.
- **승리**: 배열 내 모든 블록의 상태가 `removed`가 되면 레벨 클리어.
- **패배**: 생명(Lives)이 0이 되면 게임 오버. (재도전 시 동일한 보드와 블록 구조를 그대로 초기화하여 제공)
- **힌트 시스템**: 현재 `isBlockFree`가 true인 블록들을 모두 찾은 뒤, 그중 무작위로 하나를 선택하여 하이라이트(시각적 효과) 처리합니다.
